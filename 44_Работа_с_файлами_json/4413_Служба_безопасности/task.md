# Служба безопасности

*   Ограничение времени: 1 секунда
*   Ограничение памяти: 64Mb
*   Ввод: `run.py`
*   Вывод: стандартный вывод или `output.txt`

После успешно отраженной хакерской атаки на сервер вашей организации могли быть повреждены или заменены некоторые записи в файлах с данными. Хорошо, что у вас остались резервные копии. Теперь вам ничего не стоит проверить правильность данных клиента.

Напишите функцию `check_security(name, place)`, которая получает кортеж с фамилией, именем и отчеством клиента и регионом, проверяет корректность ИНН и возвращает кортеж из двух значений: корректен ли регион, совпадает ли контрольная сумма в ИНН. Если всё верно, оба значения равны 0, если что-либо неверно, то это значение -1. Если такого клиента в базе нет, оба значения равны `None`.

Как нужно проверять. Вашему решению будут доступны 2 файла: `taxpayer_in.json`, в котором записан список словарей на каждого клиента (ключи `lastname`, `firstname`, `middlename`, `tin`), где `tin` – это ИНН, и `regions.csv` (код региона (`region_code`), название региона (`region_name`), разделитель точка с запятой).

ИНН физического лица состоит из 12 цифр. первые две – код региона. Последние две – контрольные, считаются так:

1-я контрольная цифра: вычислить сумму произведений цифр с 1 по 10-ю на коэффициенты – 7, 2, 4, 10, 3, 5, 9, 4, 6, 8, затем найти младший разряд остатка от деления полученной суммы на 11.

2-я контрольная цифра: вычислить сумму произведений цифр с 1 по 11-ю на коэффициенты – 3, 7, 2, 4, 10, 3, 5, 9, 4, 6, 8, затем найти младший разряд остатка от деления полученной суммы на 11.

Сравнить 1-ю контрольную цифру с 11-й в ИНН, вторую с 12-й. Если они равны, то ИНН верный.

## Пример 1

**Ввод:**

```python
name = ('Корнейчук', 'Агафон', 'Фомевич')
place = 'Тульская область'

code, checksum = check_security(name, place)

print(code, checksum)
```

**Вывод:**

```
0 0
```
